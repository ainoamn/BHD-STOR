## 1) ما هو المشروع (حقيقة واحدة)

**BHD-STOR** = منصة تجارة إلكترونية عمانية متعددة البائعين (B2B/B2C/C2C/hybrid)، **ليس** منصة عقارات.

| الطبقة | التقنية |
|--------|---------|
| Frontend | Next.js 14 App Router · next-intl (ar/en RTL) · Tailwind · React Query · Zustand |
| Backend | NestJS · TypeORM (**ليس Prisma**) · PostgreSQL · Redis · Bull (ليس BullMQ) |
| النمط | Modular Monolith |

جاهزية تقريبية: **~40–55%** إنتاج · UI أعلى · مسار بيع حقيقي أقل.

## 2) ما نوقش في هذه المحادثة (ملخص القرارات)

### أ) تشغيل ومسارات
- التشغيل من `C:\dev\bhd-app` / `C:\dev\bhd-run` → ASCII فقط.
- Downloads والمسارات العربية = نسخ احتياطية، لا تشغيل Next منها.

### ب) مراجعات المساعدين الخارجيين
| المصدر | الحكم |
|--------|--------|
| **ChatGPT** | صحيح جوهرياً بعد توفر الملفات؛ أخطأ مبكراً عن «مستودع فارغ»؛ نقاط CD/API صحيحة وأُصلحت |
| **DeepSeek** | تقرير أول جيد ~75–85%؛ رد المتابعة = إعادة لصق نقد Cursor وتأخر عن الدفع |
| **جيميني / كيمي** | **خاطئ** — افترض عقارات BHD-OM بعد فشل جلب GitHub؛ **لا يُنفَّذ** |
| **Cursor** | راجع الكود محلياً؛ نفّذ P0؛ أرشف التقارير |

النصوص الحرفية محفوظة في `docs/external-reviews/`.

### ج) ميزة تنافسية معتمدة: باركود / متسلسل المتجر
- عند فتح متجر يُولَّد `storeSerial` (مثل `BHD26-A1B2C3`) + `storeCode`.
- المسح/الرابط `/ar/s/{serial}` يفتح **متجر البائع فقط**، ليس السوق العام.
- لوحة البائع وصفحة المتجر العامة: تحميل QR / باركود / ملصق / ملف نصي + طباعة.
- موثّق في `ROADMAP.md` §9.

## 5) كيف تكمل على جهاز آخر (Checklist)

```bat
git clone https://github.com/ainoamn/BHD-STOR.git
cd /d C:\dev\bhd-app
git pull origin main
```

1. اقرأ **هذا الملف** + `ROADMAP.md` + `docs/FULL_AUDIT.md`.  
2. انسخ `backend/.env.example` → `.env` و`frontend/.env.example` → `.env.local`.  
3. `PUBLIC_APP_URL` / `NEXT_PUBLIC_APP_URL` لمسارات QR.  
4. `NEXT_PUBLIC_API_URL=http://localhost:3001/api/v1`  
5. Backend: `npm install` → `npm run migration:run` → `npm run seed` → `npm run start:dev`  
6. Frontend: `npm install` → `npm run dev` → `http://localhost:3000/ar`  
7. لا تشغّل من مجلد عربي.
