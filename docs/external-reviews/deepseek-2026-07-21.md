## 1. طبيعة المشروع وحالته الحالية

BHD-STOR هو كود منصة تجارة إلكترونية عمانية متعددة البائعين (Multi-Vendor Marketplace)، مكتوب بـ Next.js 14 (Frontend) و NestJS (Backend) مع PostgreSQL و Redis.

⚠️ تنبيه مهم: المشروع ليس منتجاً جاهزاً للإنتاج، بل هو هيكل احترافي واسع (Scaffold) يضم ~680+ ملف مصدر. جاهزيته للإطلاق التجاري الفعلي تُقدّر بـ 40-55%.

## 2. نقاط الضعف والثغرات الأمنية (التشفير والحماية)

| API Keys Module | ✅ موجود |

### ❌ الثغرات ونقاط الضعف الحرجة

الأمان التشغيلي ضعيف - الحمايات موجودة في الكود لكن غير مربوطة فعلياً بشكل يضمن عملها في الإنتاج.

ENCRYPTION_MASTER_KEY غير مفعّل - مذكور في ROADMAP كخلية حرجة.

MFA/OAuth - مذكورة في الكيانات ولكن ليست مفعّلة كمنتج.

عدم تناسق التوثيق الأمني - التوثيق القديم يبالغ في الادعاءات (مثل "SOC2") بينما لا يوجد امتثال مثبت.

مسارات عربية تكسر Webpack - المشكلة مع Next.js على Windows (C:\dev\bhd-app فقط).

ORM الفعلي TypeORM - بينما بعض التوثيق يذكر Prisma.

## 3. ترابط الصفحات وأدوار المستخدمين ولوحات التحكم

### الأدوار (مبعثرة وغير متسقة)

| المصدر | الأدوار |
|--------|---------|
| database/entities/user.entity | customer, seller, admin, super_admin |
| users/entities/user.entity | customer, seller, admin, moderator |
| Frontend types | customer, vendor, admin, super_admin |
| Demo auth | customer / admin |

🚨 إشكالية: لا يوجد دور driver أو A2B agent كـ UserRole.

### مصفوفة اللوحات - مستويات الحماية

| الدور | اللوحة | مستوى الحماية |
|-------|--------|----------------|
| Customer | متجر + /orders + /wishlist | Cookie فقط؛ بيانات غالباً Demo |
| Seller | /dashboard/store | فحص isSellerRole |
| Admin | /dashboard/admin/* | حماية جزئية - بعض الصفحات بلا فحص دور |
| Logistics | /dashboard/logistics/* | ⚠️ بلا فحص دور - أي مستخدم مسجّل يمكنه الوصول |
| B2B Portal | /shipping-portal/* | مفتاح API (عميل)؛ middleware مفتوح |

Middleware يتحقق فقط من وجود cookie، وليس من صلاحية الدور.

## 5. الملاحظات التطويرية والخطة المستقبلية

### البنية التقنية الحالية

Frontend: Next.js 14 App Router + next-intl (RTL) + Tailwind + React Query + Zustand

Backend: NestJS + Swagger + ValidationPipe

ORM: TypeORM (وليس Prisma)

نمط التنظيم: Modular Monolith (Nest modules) - ليس Microservices

### التطوير المطلوب (حسب ROADMAP)

توحيد الأدوار في جميع المصادر (backend entities, frontend types, middleware)

تفعيل الحمايات - ربط الـ Guards عالمياً وليس فقط في الكود

إكمال رحلة الدفع والطلب - أولوية قصوى

إصلاح مشكلة المسارات العربية في Next.js Webpack

ترقية Queue من Bull إلى BullMQ

إزالة Feature-creep - التركيز على MVP قبل الإضافات غير الأساسية

تفعيل MFA/OAuth بشكل منتج

إعداد ENCRYPTION_MASTER_KEY للإنتاج

## 6. مقارنة الميزات: BHD-STOR vs أمازون / علي إكسبريس / علي بابا

| الميزة | BHD-STOR (حالياً) | أمازون | علي إكسبريس | علي بابا |
|--------|-------------------|--------|-------------|----------|
| Multi-Vendor | ✅ مدمج | ✅ | ✅ | ✅ (B2B) |
| B2B/B2C/C2C | ✅ مصمم | ✅ | ❌ (B2C فقط) | ✅ (B2B أساساً) |
| اللغات | العربية/الإنجليزية (RTL/LTR) | متعددة | متعددة | متعددة |
| العملات | GCC + USD/EUR/GBP | متعددة | USD أساساً | USD أساساً |
| الاشتراكات + العمولة | ✅ مصمم | ✅ (Seller Central) | ❌ | ❌ |
| الشحن واللوجستيات | ✅ مدمج (Carrier adapters) | ✅ (FBA) | ✅ | ✅ |
| AI / مساعد ذكي | ✅ Scaffold (يحتاج مفاتيح) | ✅ (Alexa, Rufus) | ✅ | ✅ |
| WhatsApp Integration | ✅ Scaffold | ❌ | ❌ | ❌ |
| لوحات تحكم متعددة | ✅ (Seller/Admin/Logistics) | ✅ | ✅ | ✅ |
| PWA / Mobile | ✅ Scaffold (غير مختبر) | ✅ | ✅ | ✅ |
| نظام الولاء | ✅ مصمم | ✅ (Prime) | ✅ | ❌ |
| حجم السوق | محلي (عمان) | عالمي | عالمي | عالمي (B2B) |
| البنية التحتية | Modular Monolith | Microservices | Microservices | Microservices |
| جاهزية الإنتاج | ~40-55% | 100% | 100% | 100% |

### نقاط القوة النسبية لـ BHD-STOR

تركيز محلي - دعم كامل للغة العربية (RTL) والعملات الخليجية

تكامل واتساب - ميزة تنافسية في السوق العماني

نموذج اشتراك + عمولة - مرونة في تحقيق الإيرادات

تنوع الأدوار (B2B/B2C/C2C/hybrid)

طبقات أمنية متعددة في التصميم

### نقاط الضعف مقارنة بالمنافسين

غير جاهز للإنتاج - بينما المنافسون منصات ناضجة

حجم السوق محدود - عمان vs عالمي

البنية - Modular Monolith vs Microservices (قابلية التوسع أقل)

الاختبار - غير مختبر كمنتج

التكاملات - تحتاج مفاتيح حقيقية وتفعيل

التوثيق غير المتسق - يبالغ في بعض الادعاءات
